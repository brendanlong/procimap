 1  """  
 2  The ProcImap package fulfills two purposes:  
 3   
 4  a) provide a reasonably simple framework for writing filters on an IMAP 
 5     Mailbox, inspired by procmail recipes 
 6   
 7  b) provide a nice object oriented interface to IMAP accounts that integrates 
 8     well with the standard library 
 9   
10  Traditionally, many people have used procmail (http://www.procmail.org) to 
11  organize their emails. They pull their email in from one or more POP servers 
12  (or receive it directly on their machine) and then filter the incoming mail 
13  through a series of procmail "recipes". The procmail recipes can deliver the 
14  mail to arbitrary mail folders (based on header data), put it through a spam 
15  filter (e.g. spamassassin), modify the message (e.g. fix and empty subject 
16  line), separate attachments, or pretty much anything else imaginable. 
17   
18  More recently, access to email over IMAP servers has become rather common and 
19  convenient. IMAP has the huge benefit of allowing people to keep their email on 
20  a central server and access it from everywhere with multiple clients. However, 
21  in many cases, users don't have the ability to use procmail together with an 
22  IMAP server.  In some cases, there are other means of filtering incoming 
23  messages. For example, Google's popular GMail service allows users to define 
24  some filters. These solutions generally don't have the full power of procmail; 
25  it is usually not possible to modify messages. 
26   
27  To fill this gap, I decided to write a python framework that would make it easy 
28  to have "recipes" for incoming mail messages. The idea is to have an IMAP 
29  client that logs into the server and continuously monitors the INBOX for new 
30  messages. These are then run through a series of recipes, and may be moved to 
31  different mailboxes (on the same server or not), or be changed (i.e. replaced 
32  by a modified version) 
33   
34  This framework is provided by the ProcImap module. In ProcImap.ProcImap you 
35  will find the AbstractProcImap class. To write filters for your IMAP inbox, you 
36  have to subclass from this abstract class and fill it with "recipes". Calling 
37  the 'run'-method of your derived class will then start to monitor and filter 
38  your IMAP account. For details, see the documentation of the ProcImap.ProcImap 
39  package.   
40   
41  While writing the ProcImap.ProcImap class, I felt that python's IMAP interface 
42  was rather weak, providing only the imaplib module, which is basically just a 
43  low level wrapper around the IMAP protocol. Specifically, IMAP was not integrated 
44  in the 'mailbox' and 'email' packages of the standard library. 
45   
46  [...] 
47   
48  To summarize, the ProcImap package has the following structure: 
49   
50  First, for the part that allows you to write filters: 
51  - The ProcImap module, which provides an abstract class from which you can 
52    derive to write you own filters 
53  - The Utils module, which contains some classes and functions that I 
54    thought were helpful in the context of writing filters. For example there 
55    is a class that makes it easy to have blacklists/whitelists 
56   
57  Second, and independently, there is the object oriented IMAP interface: 
58  - The ImapServer module containing the ImapServer class, which is a thin 
59    layer over the imaplib module of the standard library or the imaplib2 module 
60    provided by Piers Lauder. 
61  - The ImapMailbox and ImapMessage modules, which integrate into the standard 
62    library. 
63   -The MailboxFactory module, which just makes organizing your IMAP accounts 
64    easier: You can define the account data (with all the clear-text-passwords) in 
65    a central configuration file, and then create ImapMailbox objects on the fly 
66    based on that data. 
67  The ImapMailbox class is the "frontend" class, which you will be dealing 
68  with most of the time. 
69  Lastly, the  imaplib2 module is included, which was written by Piers 
70  Lauder. It is a direct replacement of the imaplib module in the standard 
71  library, and is used as a backend for my ImapServer class by default. Since 
72  the original purpose of the ProcImap package was to create clients that 
73  continuously monitor your INBOX, the IMAP 'idle' command was very useful. 
74  Unfortunately, the idle command is not supported by the imaplib library, but 
75  only by the multi-threaded imaplib2. On some systems, I have seen problems with 
76  the imaplib2 library (lock-ups of the connection). If you experience problems, 
77  you can set a flag in the source code of the ImapServer module to use the 
78  standard imaplib library. See the documentation of the ImapServer module 
79  for details. 
80  """
